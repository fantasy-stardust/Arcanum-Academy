# Naming Conventions | 命名规则

The guild names things so that responsibility can be traced without argument.

公会命名事物，不是为了华饰，而是为了让责任可以被清楚追索。

---

## Construct Names | 构造命名

Use the form `purpose-form-mark` when naming a complete construct.

完整构造物建议采用 `用途-形态-标记` 的形式命名。

Examples:

示例：

- `gate-sentinel-mk1`
- `lantern-lift-mk2`
- `study-timer-mk1`

## Hall Prefixes | 分殿前缀

Use hall prefixes when marking parts owned by a specific hall.

标记由特定分殿负责的部件时，建议使用分殿前缀。

- `GLM` for golemcraft 信息技术
- `CRC` for circuitry 电子
- `MCH` for mechanica 机械

## Interface Names | 接口命名

Use the form `source-to-target-purpose`.

接口命名建议采用 `来源-to-目标-用途` 的形式。

Examples:

示例：

- `glm-to-crc-control`
- `crc-to-mch-drive`
- `mch-to-glm-feedback`

## Revision Marks | 修订标记

Use increasing revision marks such as `r1`, `r2`, `r3` whenever the change affects integration.

凡改动会影响整合关系时，应使用递增修订标记，如 `r1`、`r2`、`r3`。

## Principle | 原则

Names should reveal purpose before ornament.

命名应先揭示用途，再容许修饰。
