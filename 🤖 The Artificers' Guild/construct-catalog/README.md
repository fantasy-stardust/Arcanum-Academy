# Construct Catalog | 造物图鉴

This first volume is thin by design: the guild would rather keep a short catalog of named examples than a swollen shelf of unproven boasts.

此首卷刻意保持薄册：公会宁可保留一本简短而具名的范例图鉴，也不愿堆满未经验证的夸口之物。

---

## Catalog Charge | 图鉴之责

The Construct Catalog gathers typical builds, planned entries, and instructive failures so apprentices can see not only what the guild admires, but also what it refuses to forget.

造物图鉴收纳典型造物、待造条目与教训性失败案例，让学徒看见的不只是公会所赞许之物，也包括公会决意不忘之失。

## Axes of Reading | 阅读轴线

Each entry may be read by four axes:

每个条目可沿四条轴线阅读：

- Record state 入档状态
- Hall configuration 分殿组合
- Function domain 功能类型
- Build complexity 构造复杂度

若你需要知道下一步该尝试什么，应先读 [../task-board/README.md](../task-board/README.md)。若你需要知道在公会语言里，完成品、待造品与警示品各自长什么样，则应先读此图鉴。

## Catalog Saying | 图鉴短谚

The guild remembers by naming, and it names only what can teach.

公会以命名来记忆，而它只命名那些能够授人以教训之物。
