# Project Record Format | 项目记录格式

The guild record is not a memory aid for its author alone; it is a chain another apprentice must be able to follow.

公会档案并不是只给原作者本人补记忆的线索，而是一条后来学徒也必须能够循迹而行的链条。

---

## Record Charge | 格式之责

Use this format when a construct is being prepared for proof, revision, archival reuse, or catalog entry.

当一件构造准备进入受试、修订、归档复用或图鉴编目时，应遵循此格式。

## Required Record Spine | 必备记录主链

Each project record should preserve the following sequence:

每份项目记录都应保存以下顺序：

1. Brief or declared purpose 简报或已声明用途
2. Participating halls and owned parts 参与分殿与各自负责部分
3. Interfaces, handoff notes, and shared conditions 接口、交接说明与共用条件
4. Safety position, tool use, and measurement basis 安全状态、器具使用与测量基准
5. Proof plan and observed result 受试计划与实际结果
6. Revision marks and present state 修订标记与当前状态

## Required Headings | 必备标题

Every project preserved for guild use should declare at least these headings:

凡准备纳入公会使用体系的项目，至少应声明以下标题：

- Construct Name 构造名称
- Purpose 用途
- Participating Halls 参与分殿
- Core Interfaces 核心接口
- Required Tools 所需器具
- Borrowed Tools and Custody 借用器具与保管
- Safety Position 安全状态
- Measurement Baseline 测量基准
- Source Records 来源记录
- Proof Plan 受试计划
- Known Risks 已知风险
- Revision Notes 修订记录
- Record Status 入档状态

## Status Words | 状态词

Status words must match the guild record states defined in [README.md](README.md):

状态词必须与 [README.md](README.md) 中定义的公会档案状态一致：

- `current`
- `archived`
- `superseded`
- `failed-but-instructive`

## Revision Marks | 修订标记

Revision marks should proceed in short steps such as `r1`, `r2`, and `r3`. If a record changes status, write both the new status and the reason for the change.

修订标记应按 `r1`、`r2`、`r3` 这类短步推进。若记录状态发生变化，应同时写出新状态与变化原因。

## Minimum Supporting Leaves | 最低附带卷叶

No record is considered complete unless it can point to the following supporting leaves when applicable:

凡记录在适用处不能回指以下卷叶者，不视为完整：

- Interface notes or templates 接口说明或接口模板
- Test record or proof result 受试记录或验证结果
- Handoff note 交接说明
- Named measurement range 已命名测量范围
- Lessons kept after revision 修订后留下的教训

## Closing Rule | 结尾规则

If a later apprentice cannot tell what was built, what was measured, and why the guild trusted it, then the record is unfinished.

若后来的学徒仍看不出此物究竟造了什么、测了什么，以及公会为何信任它，那么这份记录便仍未写完。
