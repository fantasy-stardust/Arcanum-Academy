## Proof Sequence | 受试顺序

1. Draft the build in [../joint-constructs/project-template.md](../joint-constructs/project-template.md). 先用 [../joint-constructs/project-template.md](../joint-constructs/project-template.md) 起草项目记录。
2. Name interfaces and handoff conditions with the design-vault leaves. 再用设计密库卷叶写清接口与交接条件。
3. Select the chamber protocol that best matches the first likely weakness. 选择最贴近首要薄弱点的试验规程。
4. Record the result in the proving format, even if the result is failure. 无论成败，都按试验格式记录结果。
5. Promote the construct toward the catalog only when another apprentice could repeat the route. 唯有当另一位学徒也能复走此路时，才可将其推进图鉴。

## Chamber Watch | 试验警示

If a build passes once but cannot explain why, the board sends it back for clearer notes before honoring it as progress.

若一件造物虽然通过一次，却说不清自己为何通过，任务板会先把它送回补写说明，而不会把它当作真正进展。
