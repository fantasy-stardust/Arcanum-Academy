# Code to Circuit | 代码到电路

When logic crosses into circuitry, it must stop being abstract and become explicit timing, state, or command.

当逻辑跨入电子层时，它就不能再停留在抽象之中，而必须化为明确的时序、状态或指令。

---

- Name the output state before wiring begins 在接线前先命名输出状态
- Record expected voltage, pulse, or signal form 记录预期电压、脉冲或信号形式
- Ensure a circuit apprentice can test the behavior without reading source code alone 确保电子学徒不必只靠阅读代码也能验证行为
- Never hide timing assumptions inside undocumented defaults 不可把时序假设藏在未记录的默认值里
